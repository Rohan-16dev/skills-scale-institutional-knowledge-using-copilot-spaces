# Process Doc Update — Implementation Checklist

Purpose: a lightweight, repeatable checklist teams can use when updating or adding program/process documentation in docs/.

- [ ] Create an issue using .github/ISSUE_TEMPLATE/add-update-content-to-process-docs.yml and link stakeholders
- [ ] Draft proposed content in a branch under docs/ (link branch in the issue)
- [ ] Assign a Documentation Owner for the change and add them as reviewer
- [ ] Review proposed changes with PM/PdM and at least one engineering lead
- [ ] Incorporate feedback and update docs/ files (include examples or checklists when helpful)
- [ ] Run quick readability check and confirm links and templates work
- [ ] Merge changes after approval and close the issue; add a short note in the release or weekly update
- [ ] Schedule a re-review date (e.g., quarterly) and add to the Documentation Owner's calendar

Suggested file path: docs/process-templates/process-doc-update-checklist.md
